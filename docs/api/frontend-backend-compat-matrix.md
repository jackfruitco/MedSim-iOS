## High-priority residual gaps

| Area | Confirmed residual gap |
| --- | --- |
| generic ChatLab breadth | Generic simulation/chat coverage is still thinner than backend breadth: the app now matches the contract better, but it is not a full frontend for every generic conversation/tool/lab-order workflow. |
| trainerlab event families | Backend-backed event families for problems, assessment findings, diagnostic results, resources, and disposition still need clean SwiftUI entry points. |
| generic lab-orders UX | The generic asynchronous `/lab-orders/` endpoint is modeled in code but not yet exposed as a dedicated mobile workflow. |
